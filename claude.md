# ------------------------------------------------------------
# 1. 任务执行规则
# ------------------------------------------------------------
task_rules:
  - name: "分析阶段"
    description: "每次任务开始前，先分析是否需要读取当前提及文件之外的额外文件。如果需要，先向我确认，我同意后再继续。"
    priority: highest

  - name: "严格按指令操作"
    description: "只修改我明确要求修改的内容。未提及的地方不要改动（包括格式、样式、注释、缩进等）。"
    priority: highest

  - name: "影响范围确认"
    description: "如果某个修改会影响其他页面或全局样式，先暂停并告诉我影响范围，我确认后再继续。"
    priority: high

  - name: "卡点暂停"
    description: "遇到不确定、不明确或无法执行的情况，直接暂停并告诉我问题，我来确认下一步。"
    priority: highest

  - name: "清晰反馈"
    description: "每次修改完成后，告诉我具体改了哪些文件、哪些位置、修改了什么内容。"
    priority: high

  - name: "禁止自动格式化"
    description: "不要主动格式化整个文件或项目，只修改指定区域。"
    priority: high

# ------------------------------------------------------------
# 2. 交互规则
# ------------------------------------------------------------
interaction:
  - rule: "每次响应前，先告诉我你理解的任务是什么"
  - rule: "如果我的要求不明确，先提问再行动"
  - rule: "保持单次任务聚焦，不要一次性修改多个不相关的文件"
